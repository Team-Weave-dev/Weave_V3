- 12~15: 디렉토리 목적
- 16~19: 핵심 책임
- 20~22: 구조 요약
- 23~25: 파일 라인 맵
- 26~28: 중앙화·모듈화·캡슐화
- 29~33: 작업 규칙
- 34~38: 관련 문서

## 디렉토리 목적
Supabase 데이터베이스 스키마 변화를 관리하는 SQL 마이그레이션 모음입니다.
Storage 시스템과 1:1 매핑되는 테이블과 정책을 유지합니다.

## 핵심 책임
- 테이블·뷰·정책 마이그레이션 정의
- 버전 순서와 롤백 전략 기록

## 구조 요약
- 하위 디렉토리가 없습니다.

## 파일 라인 맵
- 추적 가능한 파일이 없습니다.

## 중앙화·모듈화·캡슐화
- 데이터베이스 스키마 변경은 이 디렉터리에서만 정의하고 수동 변경을 금지

## 작업 규칙
- 새 마이그레이션 작성 후 Supabase CLI로 검증
- Storage 타입과 구조가 일치하는지 확인
- RLS 정책 변경 시 인증 문서와 동기화
