# Weave 서비스 사용자 조사 계획

## 조사 개요

### 배경
Weave는 프로젝트 중심의 대시보드 관리 서비스로, 위젯 기반의 유연한 정보 구성과 실시간 협업 기능을 제공합니다. 오프라인 사용자 테스트를 통해 실제 업무 환경에서의 유용성과 개선점을 파악하고자 합니다.

### 조사 목적
1. 프로젝트 관리 워크플로우에서 Weave의 실용성 검증
2. 위젯 시스템의 직관성 및 활용도 평가
3. 기존 프로젝트 관리 도구 대비 차별화 요소 발견
4. UI/UX 개선 사항 및 사용자 니즈 도출

### 조사 대상
- **서비스명**: Weave (프로젝트 대시보드 관리 서비스)
- **주요 기능**: 프로젝트 관리, 대시보드 위젯 (캘린더, TODO 리스트), 모바일 대응
- **타겟 사용자**: 프로젝트 관리 도구를 사용하는 지식 노동자 (프리랜서, PM, 스타트업 팀원)

### 조사 규모
- **참여자 수**: 10-15명
- **소요 시간**: 1인당 45-60분
- **조사 기간**: 2주

## 테스트 시나리오

### 시나리오 1: 신규 프로젝트 시작
**상황**: 새로운 웹사이트 리뉴얼 프로젝트를 시작한다고 가정

**태스크**:
1. "웹사이트 리뉴얼" 프로젝트 생성
2. 프로젝트 정보 입력 (기간: 3개월, 예산, 설명)
3. 주요 마일스톤 3개를 캘린더에 추가 (기획 완료, 디자인 완료, 개발 완료)
4. TODO 리스트에 초기 작업 5개 등록
5. 대시보드에서 전체 현황 확인

**관찰 포인트**:
- 프로젝트 생성 소요 시간 (목표: 2분 이내)
- 입력 필드 이해도 및 혼란 여부
- 캘린더/TODO 위젯 간 이동 패턴
- 정보 구조 파악 여부
- 저장/취소 버튼 인지 여부

### 시나리오 2: 기존 프로젝트 관리
**상황**: 진행 중인 프로젝트의 일정 및 할 일 관리

**태스크**:
1. 프로젝트 목록에서 선택
2. 완료된 TODO 3개 체크 처리
3. 새로운 일정 추가 (회의, 마감일)
4. 프로젝트 진행 상황 파악
5. 다른 프로젝트로 전환

**관찰 포인트**:
- 네비게이션 직관성
- 상태 변경 용이성 (체크박스, 드래그앤드롭 등)
- 정보 가독성 (진행률, 남은 작업 등)
- 모바일에서의 편의성
- 프로젝트 간 전환 속도

### 시나리오 3: 멀티 프로젝트 비교 및 의사결정
**상황**: 여러 프로젝트 중 우선순위를 결정해야 하는 상황

**태스크**:
1. 대시보드에서 전체 프로젝트 현황 파악
2. 마감 임박 프로젝트 확인
3. 오늘 해야 할 작업 리스트 확인
4. 우선순위 조정 (프로젝트 순서 변경)

**관찰 포인트**:
- 정보 비교 용이성 (여러 프로젝트 동시 확인)
- 긴급도/중요도 인지 여부
- 의사결정 지원 정도
- 멀티 프로젝트 관리 효율성
- 대시보드 레이아웃 만족도

## 관찰 및 평가 기준

### 정량적 지표

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 태스크 완료 시간 | 프로젝트 생성 | < 2분 |
| | TODO 추가 | < 30초 |
| | 위젯 설정 | < 1분 |
| 오류 발생 횟수 | 잘못된 버튼 클릭 | < 3회 |
| | 뒤로가기/재시도 | < 2회 |
| | 도움 요청 | < 3회 |
| 태스크 완료율 | 시나리오 1 | 100% |
| | 시나리오 2 | 100% |
| | 시나리오 3 | 80% 이상 |

### 정성적 관찰 항목

**표정 및 반응**:
- 😕 혼란스러워하는 순간 (어디를 클릭해야 할지 모름)
- 😊 만족스러운 순간 (기능 발견, 편리함 체감)
- 😤 좌절/불편함 표현 (에러, 예상과 다른 동작)
- 🤔 고민하는 순간 (선택지 비교, 기능 탐색)

**언어 표현 기록**:
- "어떻게 하지?" → UI 개선 필요
- "이게 뭐지?" → 기능 설명 부족
- "오, 좋네요!" → 만족 포인트
- "이상한데..." → 버그 또는 UX 문제

**행동 패턴**:
- 머뭇거림 (3초 이상 멈춤) → 직관성 부족
- 빠른 클릭/확신 → 직관적 UI
- 탐색 패턴 (클릭 순서, 메뉴 탐색)
- 숨겨진 기능 발견 여부

### 만족도 평가 (5점 척도)

| 평가 항목 | 1점 (매우 불만) | 3점 (보통) | 5점 (매우 만족) |
|----------|----------------|-----------|----------------|
| 전체 만족도 | 사용 의향 없음 | 일부 개선 필요 | 바로 사용 가능 |
| UI 직관성 | 혼란스러움 | 학습 후 사용 가능 | 즉시 이해 |
| 기능 유용성 | 불필요한 기능 | 기본 기능 충족 | 필수 기능 완비 |
| 학습 용이성 | 어려움 | 보통 | 매우 쉬움 |
| 실사용 의향 | 전혀 없음 | 고려 중 | 확실히 사용 |
